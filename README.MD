# llm2rtf

Конвертер математического Markdown нейросетей (Gemini/Google AI Studio и др.) в формат RTF, который корректно отображается в Word, WordPad и других текстовых редакторах. C **сохранением формул** (в упрощенном виде), **таблиц** и **форматирования**.
### Для чего нужен?
Нейросети выдают математические ответы в формате Markdown и LaTeX. При обычном копировании в редакторы типа Word, особенно старых версий, формулы превращаются в код (LaTex), а таблицы и заголовки теряют структуру (md).

Скрипт берет текст из буфера обмена, вычищает мусор LaTeX, конвертирует сложные формулы в символы Unicode и упаковывает всё в Rich Text Format (RTF), который поддерживается практически всеми редакторами текста.

*Вы получаете не программный код, а человекочитаемые формулы с использованием греческих букв, степеней и математических знаков.*

**Исходный текст от LLM:**
> В $\triangle ABC$ известно: $\angle \alpha = 60^\circ$, сторона $AB = \mathbf{10\sqrt{3}}\text{ см}$. Найти высоту $h = \frac{AB \cdot \sin(\alpha)}{\sqrt{x^2 + y^2}}$.

Результат при копировании из LLM в редактор **без конвертации**:
> `В $\triangle ABC$ известно: $\angle \alpha = 60^\circ$, сторона $AB = \mathbf{10\sqrt{3}}\text{ см}$. Найти высоту $h = \frac{AB \cdot \sin(\alpha)}{\sqrt{x^2 + y^2}}$.`

### Какие редакторы поддерживаются
RTF — это стандарт форматирования, уходящий корнями в мохнатый 1987 год. Результат работы скрипта можно вставлять в:
- Microsoft Word (любых версий, включая 97, 2003, 2007 и современные).
- WordPad
- LibreOffice / OpenOffice.
- Старые редакторы типа AbiWord.
- Текстовые процессоры на macOS (TextEdit) и Linux.

### Как это работает
1. Вы копируете текст из окна чата с нейросетью (`Copy as markdown`).
2. Запускаете скрипт `llm2rtf.py` (двойным кликом, иконкой на панели, горячей клавишей или через терминал: `python llm2rtf.py` - тут как придумаете).
3. Скрипт обрабатывает текст и обновляет буфер обмена.
4. Вы вставляете результат в свой редактор (`Ctrl+V`).

## Что скрипт умеет
- [x] Автоматическая очистка мусорных тегов LaTeX (`\left`, `\quad`, `\begin`).
- [x] Превращение дробей (`\frac{a}{b}`) и вложенных корней (`\sqrt`) в читаемый Unicode-вид (`√`).
- [x] Математические символы: заменяет команды типа \alpha, \triangle, \perp на α, △, ⊥.
- [x] Конвертация индексов и степеней в надстрочные Unicode-символы (`x^2` в `x²` и `x_{12}` в `x¹²`.).
- [x] Минимальный Markdown (Заголовки, **Жирный**, *Курсив*). Математические переменные автоматически становятся курсивом, три уровня заголовков.
- [x] Преобразует Markdown-таблицы в таблицы RTF (с рамками и авто-шириной)
- [x] Мультипоточный буфер обмена для Windows (одновременная отправка Plain Text и RTF).

### Ограничения
Инструмент ориентирован на быструю подготовку текста и использует "линейную" запись формул. Сложная многоэтажная верстка упрощается для сохранения совместимости со старыми редакторами. Например: интегралы, суммы с индексами сверху/снизу и матрицы не отрисовываются графически.

**Примеры упрощений:**
1. **Дроби**: Сложные дроби записываются через слеш.
   - Было: `\frac{a + b}{c + d}`
   - Стало: `(a + b) / (c + d)`
2. **Корни**:
   - Было: `\sqrt{x^2 + y^2}`
   - Стало: `√(x² + y²)`

## llm2rtf v2.0

Вторая версия скрипта (`llm2rtf2.py`) переходит от линейного текста к **полноценным математическим формулам**. Первая версия упрощала конструкции (превращая дроби в строку через слэш `(a)/(b)`), чтобы 100% гарантировать работу в любых, даже самых примитивных полях ввода. Версия 2.0 использует нативные математические поля `EQ` (Equation fields), встроенные в движок Microsoft Word.

### Главные отличия v2.0:
- [x] **Настоящие двухэтажные дроби:** `\frac{a}{b}` теперь рендерится как классическая вертикальная дробь, а не в строку.
- [x] **Матрицы и системы уравнений:** Поддержка окружений `\begin{matrix}` и `\begin{cases}`.
- [x] **Корни с индексами:** Поддержка синтаксиса `\sqrt[n]{x}`.
- [x] **Умный парсер "изнутри-наружу":** Скрипт сворачивает глубоко вложенные формулы без построения тяжелых AST-деревьев (абстрактных синтаксических деревьев).
- [x] **Нативный буфер macOS/Linux:** RTF-код теперь кладется напрямую в буфер обмена через `osascript` (Mac) и `wl-copy`/`xclip` (Linux), без создания временных файлов-костылей на диске.
- [x] **Безопасный рефакторинг:** Убрано дублирование кода, улучшена обработка невидимых скобок (`\left.`).

*Техническое ограничение (почему эмодзи заменяются на `?`):* В обычный буфер обмена эмодзи копируются прекрасно, но стандарт RTF требует строгой 16-битной кодировки (`\uXXXX`). Символы с кодом больше 65535 (как эмодзи) требуют программного вычисления "суррогатных пар" (разбиения на два кода). Чтобы не раздувать парсер и не вызывать краш старых версий Word, скрипт безопасно конвертирует их в знак вопроса.
